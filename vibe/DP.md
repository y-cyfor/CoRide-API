# CoRide-API 需求池 (DP)

> 基于 PRD.md 需求文档与实际代码审计，对比当前实现，整理剩余功能缺陷和新增建议。
> 已完成项归档至 `DP-completed.md`。
> 最后更新: 2026-04-17

---

## 一、暂不实施的功能

| 编号 | 功能 | 排除原因 |
|------|------|----------|
| 10 | 系统设置页面代理配置编辑 | 需后端新增 PUT 接口，个人项目优先级低 |
| 11 | 配置导入导出 | 个人项目，备份需求低 |
| 12 | 批量操作 | 体验优化，非核心功能 |
| 14 | 渠道 API Key 轮询可视化 | 需后端新增轮询状态追踪接口，改动大 |

## 二、不应纳入的功能

| 编号 | 功能 | 排除原因 |
|------|------|----------|
| X1 | `/v1/embeddings` 等非对话端点 | 项目定位为 LLM 对话 API 中转 |
| X2 | 多租户/多组织隔离 | 目标用户为个人/小团队 |
| X3 | 支付/计费系统 | 非商业化项目 |
| X4 | SSO/OAuth 第三方登录 | PRD 明确"无注册，管理员手动创建" |
| X5 | Redis 集群 | PRD v2.x 规划 |
| X6 | 插件系统 | PRD v2.x 规划 |

## 三、待实现功能

### 优先级 A（高价值 + 可实现）

| 编号 | 功能 | 说明 | 状态 |
|------|------|------|------|
| 20 | 渠道用量统计面板 | 中转层按 token/请求数统计每个渠道的实际消耗，在渠道管理页显示累计用量、今日用量、趋势。纯统计，不依赖上游 API | 待开发 |
| 21 | 请求缓存 | 中转层对相同请求做短期缓存（可配置 TTL），同时保留上游缓存 header（如 `anthropic-beta: prompt-caching-3`），不破坏上游缓存命中。命中缓存直接返回，零 token 消耗 | 待开发 |
| 22 | 渠道配额可视化 | 在渠道用量面板上叠加"已用/总限"进度条，直观显示渠道配额剩余。复用现有渠道配额字段，补充可视化 | 待开发 |

### 优先级 B（有价值 + 需要设计）

| 编号 | 功能 | 说明 | 状态 |
|------|------|------|------|
| 23 | API Key 独立配额（渠道级） | 扩展 `quotas` 表增加 `channel_id` 字段，支持"用户 × 渠道"维度的配额限制。`channel_id IS NULL` 为用户级总配额（兜底），`channel_id = N` 为该用户在特定渠道上的独立配额。优先级：渠道配额 > 用户总配额 | 待开发 |
| 24 | IP 黑白名单 | 全局白名单 + Key 级别白名单，简单 CIDR 匹配，不涉及地理位置解析 | 待开发 |
