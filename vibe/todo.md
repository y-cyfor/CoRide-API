# LiteProxy 功能优化 & 新增清单

> 最后更新: 2026-04-15

## 一、PRD 已规划但未实现/需完善的功能

### 1. Streaming 流式透传
- **位置**：`backend/src/service/proxy.rs:133`
- **现状**：读取完整响应后一次性返回，SSE 请求会阻塞
- **PRD要求**：支持 Streaming 响应透传
- **实现**：使用 `resp.bytes_stream()` 逐 chunk 转发 SSE 流

### 2. 请求/响应体日志记录
- **现状**：`config.yaml` 有 `log_request_body/log_response_body` 配置项，但 proxy 返回的截断 body 未写入数据库
- **PRD要求**：可选记录请求体和响应体摘要
- **实现**：根据配置将截断后的 body 写入 `request_logs` 表

### 3. 渠道配额周期自动重置
- **现状**：用户配额有周期重置逻辑，渠道配额没有
- **PRD要求**：渠道配额支持 hourly/daily/weekly/monthly 周期
- **实现**：在渠道配额检查时，判断 `quota_period_end`，过期时自动重置 `quota_used = 0` 并更新周期时间

### 4. 用户表单增加模型绑定
- **位置**：`web/src/views/manage/user/index.vue`
- **现状**：无 `enabled_models` 选择 UI
- **PRD要求**：用户可绑定可用模型列表
- **实现**：增加 MultiSelect 组件，从已有模型列表中选择，存储为 JSON 数组

### 5. 用户列表展示 API Key
- **位置**：`web/src/views/manage/user/index.vue`
- **现状**：列表不显示 API Key
- **PRD要求**：显示前8位，其余 `****`
- **实现**：增加 API Key 列，支持一键复制

### 6. 用户列表展示配额使用进度
- **位置**：`web/src/views/manage/user/index.vue`
- **现状**：无配额使用展示
- **PRD要求**：配额使用进度条
- **实现**：增加配额列，显示已用/总量 + 进度条

### 7. 渠道列表显示配额进度
- **位置**：`web/src/views/manage/channel/index.vue`
- **现状**：仅显示 `quota_used` 数字
- **PRD要求**：进度条 + 文本（如 "150/1000 次/月"）
- **实现**：显示配额进度条和周期信息

### 8. 仪表盘增加统计图表
- **位置**：`web/src/views/home/` 或 `web/src/views/dashboard/`
- **现状**：仅有数字卡片（已实现基础框架）
- **PRD要求**：折线图、饼图、Top5 列表
- **实现**：
  - 近7日调用趋势折线图（ECharts）
  - 渠道用量分布饼图（ECharts）
  - 热门模型 Top5 排行

### 9. 统计页面增加筛选功能
- **位置**：`web/src/views/manage/stats/`
- **PRD要求**：按时间范围、用户、模型、渠道筛选
- **实现**：增加筛选表单，后端接口支持对应参数

### 10. 日志详情抽屉
- **位置**：`web/src/views/manage/log/`
- **PRD要求**：日志详情展示请求体/响应体 JSON 格式化、Token 明细、耗时
- **实现**：增加详情抽屉组件

### 11. 统计卡片增加 P95 耗时与错误率
- **位置**：`web/src/views/dashboard/`
- **PRD要求**：P95 耗时、错误率百分比
- **实现**：后端统计接口增加 P95 计算和错误率

### 12. 实时请求表格
- **位置**：`web/src/views/dashboard/`
- **PRD要求**：展示最新 10 条请求实时表格
- **实现**：后端返回最新日志，前端表格展示

### 13. Token 消耗堆叠柱状图
- **位置**：`web/src/views/manage/stats/`
- **PRD要求**：输入/输出 Token 分开堆叠展示
- **实现**：ECharts 堆叠柱状图，x轴时间，y轴token量

### 14. 导出 CSV 功能完善
- **现状**：仅 `/admin/logs/export` 导出日志
- **PRD要求**：`/admin/stats/export` 导出统计数据
- **实现**：新增统计数据导出接口

### 15. 系统设置页面功能完善
- **位置**：`web/src/views/manage/settings/`
- **PRD要求**：服务器配置、全局请求头、日志级别动态调整

### 16. 应用伪装预设管理完善

- **实现**：前端增加删除确认提示和标识符格式校验

## 二、新增功能建议

### 17. 用户自助配额查询
- **新增接口**：`GET /v1/user/info`
- **用途**：用户用 API Key 查询自己的配额使用情况、可用模型列表
- **返回**：用户名、配额类型、已用/总量、可用模型、状态

### 18. 渠道健康检查
- **新增**：后台定时检测渠道可用性
- **逻辑**：
  - 定时发送测试请求（如每5分钟）
  - 连续失败 N 次自动禁用渠道
  - 恢复后自动启用
- **前端**：渠道列表显示健康状态指示灯

### 19. 配额预警通知
- **新增**：配额使用达到阈值时告警
- **触发条件**：配额使用超过 80% / 90% / 100%
- **方式**：
  - 管理面板内红点提示
  - 可选 Webhook 回调通知（PRD v2.x 规划）

### 20. 配置导入导出
- **新增**：导出/导入渠道、模型、用户配置
- **格式**：JSON 文件
- **用途**：备份恢复、多实例同步配置

### 21. 批量操作
- **新增**：
  - 批量启用/禁用渠道
  - 批量删除日志（按时间范围）
  - 批量重置配额

### 22. 请求日志搜索增强
- **新增**：
  - 按时间范围筛选（开始/结束时间 DatePicker）
  - 按端点类型筛选
  - 按耗时排序
  - 关键词搜索（模型名）

### 23. 渠道快速导入预设模型
- **新增**：创建渠道时，一键导入该供应商的常见模型
- **示例**：选择阿里云渠道时，自动添加 qwen-turbo / qwen-plus / qwen-max 等模型映射

### 24. 供应商预设快速选择（增强）
- **现状**：前端已有供应商 BaseURL 下拉选择
- **增强**：PRD 要求区分普通版/CodingPlan版 + OpenAI兼容/Anthropic兼容
- **实现**：增加级联选择器：供应商 → 版本类型 → 接口类型 → 自动填充

## 三、不应纳入的功能（已排除）

| 编号 | 功能 | 排除原因 |
|------|------|----------|
| X1 | `/v1/embeddings` 向量接口 | 项目定位为 LLM 对话 API 中转，向量/图片/语音属于独立产品线 |
| X2 | `/v1/images/generations` 图片生成 | 同上 |
| X3 | `/v1/audio/transcriptions` 语音转文字 | 同上 |
| X4 | 多租户/多组织隔离 | 项目目标用户为个人/小团队，不需要多租户 |
| X5 | 支付/计费系统 | 非商业化项目 |
| X6 | SSO/OAuth 第三方登录 | PRD 明确"无注册，管理员手动创建用户" |
| X7 | Redis 集群支持 | PRD v2.x 规划，当前核心定位为 SQLite 嵌入式 |
| X8 | 插件系统 | PRD v2.x 规划，当前阶段不引入复杂度 |
